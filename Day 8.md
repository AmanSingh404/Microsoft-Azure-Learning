# Day 8 — Azure Network Security Groups

## Objective
Learn how to secure cloud infrastructure using NSG rules.

## Implementation

1. Opened VM networking settings.
2. Accessed Network Security Group.
3. Created inbound rule to block HTTP traffic.
4. Tested website accessibility.

## Rule Configuration

Port: 80  
Protocol: TCP  
Action: Deny  

## Result

Website access was successfully blocked using NSG rule.

## Key Learning

Network Security Groups act as firewalls controlling traffic to cloud resources.
